# 局域网文件传输工具 V1.4 需求文档

## 1. 版本目标

在 V1.3（中文 GUI + 手动连接控制）基础上，新增以下能力：

1. 传输过程可视化进度展示（会话级 + 当前条目级）。
2. 按钮禁用态视觉区分，发送按钮状态自动切换。
3. 文件夹断点续传能力调研，并形成可执行的分阶段方案。

## 2. 功能需求

- FR-401：发送端 GUI 显示会话总体进度（百分比、已传输字节、速度、预计剩余时间）。
- FR-402：发送端 GUI 显示当前条目进度（条目路径、条目进度条、条目已传输字节）。
- FR-403：接收端 GUI 显示接收会话进度与当前条目进度。
- FR-404：发送按钮根据状态自动切换文案和可用性。
  - 未连接：`先开启连接`（禁用）
  - 已连接但无源：`请先添加文件`（禁用）
  - 可发送：`开始发送`（启用）
  - 发送中：`发送中...`（禁用）
- FR-405：禁用按钮需有明显视觉区分（背景、边框、文本色同时变化）。
- FR-406：保留 V1.2/V1.3 传输语义与兼容能力，不破坏 `--file`、`--src` 和既有冲突策略。
- FR-407：产出《文件夹断点续传调研结论》：可行性、风险边界、协议扩展建议、实施阶段拆分。

## 3. 非功能需求

- NFR-401：GUI 进度刷新应平滑，不造成明显卡顿。
- NFR-402：核心进度计算逻辑应可测试，避免仅依赖 UI 手工验证。
- NFR-403：不引入破坏性协议升级，优先采用向后兼容字段扩展。
- NFR-404：保持跨平台（Windows/macOS/Linux）行为一致。

## 4. 断点续传调研结论

### 4.1 是否可行

可行。建议分阶段实施，先做“文件级断点续传”，再评估“块级断点续传”。

### 4.2 推荐阶段

- 阶段 A（建议先落地）：文件级断点续传
  - 目标：网络中断后可从文件偏移继续传输。
  - 必要字段：`resume_id`、`relative_path`、`offset`、`filesize`、`mtime`。
  - 接收端：维护会话日志（journal）并返回已落盘偏移。
  - 发送端：校验文件一致性后，从偏移续传。

- 阶段 B（后续增强）：块级断点续传
  - 目标：更细粒度恢复，适用于超大文件和高抖动链路。
  - 代价：状态管理复杂度显著提升，需要块索引与校验机制。

### 4.3 风险与边界

- 一致性风险：源文件在断点后被修改，必须触发全量重传或拒绝续传。
- 状态清理：会话日志需要 TTL 或清理机制，避免积累脏状态。
- 跨平台细节：路径、时间戳精度差异需统一处理。

## 5. 验收标准

1. GUI 发送和接收页面均可实时显示进度条和文本进度信息。
2. 发送按钮状态切换符合 FR-404，禁用态样式显著区分。
3. 既有 V1.2/V1.3 自动化测试通过。
4. 新增进度/状态测试通过。
5. 文档包含断点续传调研结论与分阶段实施建议。
