# 局域网文件传输工具 V1.2 需求文档

> 版本：1.2  
> 基线：V1.1（单文件传输 + 安装打包）  
> 语言：Python 3.8+  
> 协议：TCP + JSON 头部 + 二进制数据块（扩展为会话级多条目传输）

---

## 1. 目标与范围

### 1.1 核心目标

V1.2 在保持 V1.1 能力的基础上，新增：

1. 单次连接传输多文件。
2. 直接传输文件夹（递归）。
3. 保留目录结构与文件修改时间（mtime）。

### 1.2 范围内

- 多源输入（文件/目录混合）。
- 目录递归与空目录创建。
- 同名冲突策略（默认 `overwrite`）。
- 符号链接跟随（文件与目录都跟随）。
- 单条目失败后继续传输后续条目（默认 continue）。
- 传输汇总统计（成功/失败/总字节/耗时）。

### 1.3 范围外

- 断点续传。
- 内容校验和（hash）比对。
- 传输加密与鉴权。
- 增量同步。
- POSIX 权限/属主完整保留。

---

## 2. 用户决策（已确认）

1. 冲突默认策略：`overwrite`。
2. 符号链接策略：`follow`（跟随）。
3. 失败处理策略：默认继续传后续条目。
4. 时间属性：保留文件 `mtime`。

---

## 3. 功能需求

| 编号 | 功能 | 说明 |
|---|---|---|
| FR-201 | 多源输入 | `send` 支持多个源路径；每个源可为文件或目录 |
| FR-202 | 目录递归 | 目录内文件递归传输，并在接收端创建空目录 |
| FR-203 | 结构保留 | 以每个源路径的 basename 为根，保留相对目录结构 |
| FR-204 | 多条目单会话 | 一个 TCP 连接传输 N 个目录/文件条目并统一结束 |
| FR-205 | 冲突处理 | 支持 `overwrite/skip/rename`，默认 `overwrite` |
| FR-206 | 符号链接跟随 | 发送端遇到符号链接时解析目标并按目标类型传输 |
| FR-207 | 失败继续 | 单条目失败记录后继续后续条目，最终汇总返回 |
| FR-208 | mtime 保留 | 接收端写入文件后设置源文件修改时间 |
| FR-209 | 向后兼容 CLI | 保留 `--file`，内部映射为单个 `--src` |
| FR-210 | 结果汇总 | 发送端/接收端输出总数、成功数、失败数、总字节、耗时 |
| FR-211 | 符号链接环检测 | 跟随符号链接时需检测循环引用并安全跳过，记录错误 |

## 4. 命令行需求

### 4.1 发送端

```bash
fileferry send \
  --host <ip> --port <port> \
  --src <path1> --src <path2> ... \
  [--conflict overwrite|skip|rename] \
  [--continue-on-error] \
  [--timeout 10] [--chunk-size 65536]
```

兼容参数：

```bash
fileferry send --host <ip> --port <port> --file <single_path>
```

### 4.2 接收端

```bash
fileferry recv \
  --host 0.0.0.0 --port <port> \
  [--output-dir <dir>] \
  [--conflict overwrite|skip|rename] \
  [--timeout 10] [--chunk-size 65536]
```

### 4.3 参数规则

- `--src` 可重复，多源至少一个。
- `--file` 与 `--src` 互斥；若使用 `--file`，行为等价单个 `--src`。
- `--continue-on-error` 默认开启；后续可扩展 `--fail-fast`。

## 5. 协议需求（V1.2）

### 5.1 帧结构（保持统一）

每一帧仍为：

```text
[4-byte header_len][JSON header][binary payload(optional)]
```

### 5.2 消息类型

| type | payload | 说明 |
|---|---|---|
| `session_start` | 无 | 会话开始，声明协议版本与会话级参数 |
| `entry_dir` | 无 | 声明目录条目（含 `relative_path`） |
| `entry_file` | 文件字节流 | 声明文件条目（含 `relative_path/filesize/mtime`） |
| `entry_result` | 无 | 接收端返回该条目处理结果（成功/失败/原因） |
| `session_end` | 无 | 会话结束与统计摘要 |

### 5.3 关键字段

- `protocol_version`: `"1.2"`
- `relative_path`: 使用 `/` 分隔，禁止绝对路径与 `..`
- `filesize`: 文件字节数
- `mtime`: Unix 时间戳（秒）
- `conflict_policy`: `overwrite|skip|rename`
- `continue_on_error`: `true|false`

### 5.4 安全规则

- 接收端必须对 `relative_path` 规范化校验。
- 任何路径都不得逃逸 `output_dir`。
- 非法条目直接标记失败并记录错误，按继续策略决定是否终止。

---

## 6. 业务规则

1. 发送端先展开输入源（目录递归、符号链接跟随）。
2. 展开阶段需做符号链接环检测，避免无限递归。
3. 展开后形成有序条目流：目录先于其子文件发送。
4. 文件条目按流式分块发送，不整文件入内存。
5. 接收端逐条处理并回传 `entry_result`。
6. 文件写入成功后设置 `mtime`。
7. 全部条目处理后，双方输出汇总。

## 7. 非功能需求

### 7.1 兼容性

- Linux / macOS / Windows 主流版本。
- 路径协议统一 `/`，本地落盘时由 `pathlib` 适配平台分隔符。

### 7.2 性能

- 分块传输（默认 64KB）。
- 支持大目录场景下稳定运行，不因条目数量导致内存线性暴涨。

### 7.3 可靠性

- 单条目失败可继续。
- 会话级异常需有明确错误码和可读错误文本。

### 7.4 可观测性

- 关键日志：会话开始、每条目结果、会话结束汇总。

---

## 8. 测试与验收标准

### 8.1 功能验收

1. 可一次发送“多文件 + 多级目录 + 空目录”。
2. 接收端目录结构与内容一致。
3. 同名冲突下 `overwrite/skip/rename` 行为符合定义。
4. 符号链接目标可正确传输（文件/目录）。
5. 文件 `mtime` 与发送端一致（允许秒级误差）。

### 8.2 异常验收

1. 非法路径（绝对路径、`..`）被拒绝并记录。
2. 某条目权限失败时后续条目仍可继续（默认）。
3. 网络中断时给出明确失败摘要。

### 8.3 回归验收

- V1.1 单文件传输用例全部通过（`--file` 路径不破坏）。
