# 局域网文件传输工具 V1.0 需求文档

> 版本：1.0  
> 语言：Python 3.8+  
> 协议：TCP + JSON 头部 + 二进制数据块  
> 目标：实现 Linux ↔ Windows 之间的单文件传输

---

## 1. 项目背景

在日常开发或办公中，经常需要在不同操作系统的设备间（例如 Linux 服务器与 Windows 笔记本）传输文件。现有方案（U盘、网盘、微信）不够直接或受网络限制。本项目实现一个轻量级命令行工具，通过 TCP Socket 自定义协议完成跨平台文件传输，为学习网络编程和后续功能迭代（文件夹、断点续传）打下基础。

---

## 2. 核心目标

**V1.0 只完成一件事**：  
> 发送端把本地**一个文件**完整地发送到接收端，接收端按原文件名保存，内容无差错。

不要求：文件夹、进度条、校验、断点续传、加密。

---

## 3. 功能需求

| 编号 | 功能 | 描述 |
|------|------|------|
| FR-01 | 发送单个文件 | 发送端指定目标 IP、端口、本地文件路径，主动发起连接并传输 |
| FR-02 | 接收单个文件 | 接收端监听指定端口，接收文件并保存到指定目录（默认可执行文件所在目录） |
| FR-03 | 协议封装 | 每个文件使用 `[JSON头长度 + JSON头 + 二进制文件数据]` 格式传输 |
| FR-04 | 文件名保留 | 接收端保存的文件名需与发送端原始文件名一致（不含路径） |
| FR-05 | 基础错误处理 | 连接失败、文件不存在、端口被占用等情况输出明确错误信息并退出 |
| FR-06 | 命令行参数 | 发送端和接收端均通过命令行参数配置，不使用配置文件 |

---
